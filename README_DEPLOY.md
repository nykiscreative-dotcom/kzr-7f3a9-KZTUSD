# Деплой автономной версии (Telegram-бот)

Два режима — выберите один.

## Вариант A: GitHub Actions (бесплатно, рекомендую для старта)
Ежедневная сводка в Telegram в 10:30 (Астана) без сервера. Команды бота (/today и т.д.) в этом режиме НЕ работают — только push-сводки.

1. Создайте бота: в Telegram напишите @BotFather → `/newbot` → получите токен.
2. Напишите своему боту любое сообщение, затем откройте `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите `"chat":{"id":...}` — это ваш chat id.
3. Создайте **приватный** репозиторий на GitHub, загрузите содержимое папки `deploy/` (вместе со скрытой папкой `.github/`). Скопируйте в `deploy/data/` CSV-файлы из `../data/` (стартовая история).
4. В репозитории: Settings → Secrets and variables → Actions → добавьте `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
5. Вкладка Actions → «USD/KZT Daily Radar» → Run workflow (проверка вручную). Дальше — каждый день сам.

Данные и журнал коммитятся обратно в репозиторий — история накапливается.

## Вариант B: Railway / Render / VPS (полный бот с командами)
1. Те же шаги 1–2 (токен и chat id).
2. Deploy из репозитория; start command: `python main.py --loop`; переменные окружения из `.env.example` + `TZ=Asia/Almaty`.
3. Бот отвечает на команды: `/today /signal /history /performance /settings /explain /risk` и шлёт сводку в 10:30, экстренные алерты — при аномалиях уровня «высокий/критический».
4. Диск: подключите volume к `DATA_DIR`, иначе журнал сбросится при рестарте.

Стоимость: Railway ~5$/мес (Hobby), Render free-tier засыпает (подойдёт вариант A), VPS от ~3$/мес.

## Локальная проверка
```
cd deploy
pip install -r requirements.txt
cp ../data/*.csv data/          # стартовая история
python main.py                  # один прогон без Telegram (если нет токена — просто печатает сводку)
```

## Ограничения и заметки
- yfinance неофициально использует Yahoo Finance; при блокировках добавьте retry или замените на другой источник (stooq, investing) в `advisor/collector.py`.
- Контекст ставок (`CTX` в main.py: ставка НБК, ФРС, направление политики) обновляйте после заседаний, либо доверьте это Cowork-агенту — он читает новости ежедневно.
- Все рекомендации вероятностные; бот никогда не обещает доход.

## Мобильная версия (GitHub Pages) — сводка на телефоне по ссылке
После загрузки репозитория (шаги варианта A):
1. Repository → Settings → Pages → Source: **Deploy from a branch** → Branch: `main`, папка `/docs` → Save.
2. Через 1–2 минуты сводка доступна по адресу `https://<ваш-логин>.github.io/<имя-репозитория>/` — добавьте в закладки телефона («На экран Домой»).
3. Workflow обновляет страницу каждый день в 10:30 (Астана): свежий сигнал, обе темы (Apple ⇄ SpaceX, переключатель сверху), иконка 💬 открывает диалог с Claude.

Приватность: GitHub Pages бесплатен только для публичных репозиториев. В данных нет ничего личного (открытые рыночные котировки и сигналы), страница закрыта от поисковиков (noindex), а имя репозитория можно сделать неугадываемым, например `kzr-7f3a9`. Если нужна полная приватность — Cloudflare Pages (бесплатно, поддерживает закрытый доступ).

Примечание: облачная сводка собирается автоматикой (шаблонные формулировки). Полный разбор с живым комментарием консультанта — в Cowork-версии на компьютере; она же остаётся источником истины для журнала самообучения.
